# 📜 Code of Conduct

## Our Pledge

In the interest of fostering an open and welcoming environment, we as
contributors and maintainers pledge to make participation in our project
and our community a harassment-free experience for everyone, regardless of
age, body size, disability, ethnicity, gender identity and expression,
level of experience, nationality, personal appearance, race, religion,
or sexual identity and orientation.

## Our Standards

### Positive Behavior

- **Be respectful** - Treat everyone with respect
- **Be inclusive** - Welcome newcomers and different perspectives
- **Be constructive** - Provide helpful feedback
- **Be collaborative** - Work together to improve the project
- **Be patient** - Remember that people have different skill levels
- **Be welcoming** - Greet newcomers and help them get started

### Negative Behavior

- **No harassment** - No unwelcome comments or advances
- **No trolling** - No inflammatory comments to provoke conflict
- **No personal attacks** - No insults or ad hominem attacks
- **No spam** - No promotional content or irrelevant posts
- **No hate speech** - No discriminatory language or behavior

## Our Responsibilities

### Project Maintainers

Project maintainers are responsible for:

- Defining the standards of acceptable behavior
- Enforcing this code fairly and consistently
- Removing, editing, or rejecting comments, commits, issues, and PRs
- Banning contributors who violate this code
- Making sure all contributors follow these guidelines

### Contributors

Contributors are expected to:

- Follow the code of conduct in all project spaces
- Report violations to maintainers
- Cooperate with investigations when needed

## Enforcement

### Reporting

If you witness or experience violations:

1. **Direct report** - Contact maintainers via GitHub Issues
2. **Private report** - Email maintainers directly
3. **Include details** - Provide context and evidence

### Investigation Process

1. **Acknowledge** - Maintainers will acknowledge within 48 hours
2. **Review** - Investigate the incident thoroughly
3. **Decide** - Determine appropriate action
4. **Respond** - Communicate the decision to reporter
5. **Document** - Keep records of incidents

### Consequences

Depending on severity, consequences may include:

- **Warning** - Private warning with explanation
- **Reprimand** - Public reminder of conduct
- **Temporary ban** - Suspension from project
- **Permanent ban** - Permanent removal from project

## Scope

This Code of Conduct applies to:

- GitHub repositories
- GitHub discussions
- Community events (online and offline)
- Project communication channels
